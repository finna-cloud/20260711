# Story App Interface for SillyTavern

把 SillyTavern 顯示成桌面／手機皆可使用的故事 APP 介面。擴充不會取代角色卡、世界書或模型設定；它只提供另一套操作入口。

## 主要功能

- APP 首頁與目前角色封面（自動使用角色頭像）
- 聊天室、人物設定、備忘錄、聊天日誌與本機音樂頁
- 自動讀取「主要開場白＋全部替代開場白」
- 點擊選項後切換 SillyTavern 第一則訊息的開場 swipe
- 每個角色可以分別設定開場按鈕名稱
- 手機安全區域、桌面寬螢幕與深色介面支援
- 可選擇新聊天時自動打開 APP

## 安裝

### 方法一：解壓縮安裝

1. 解壓縮下載的 ZIP。
2. 將整個 `st-app-interface` 資料夾放入：

   `SillyTavern/data/<你的使用者名稱>/extensions/`

3. 重新啟動 SillyTavern，並重新整理瀏覽器。
4. 開啟「擴充功能 → 管理擴充功能」，確認 `Story App Interface` 已啟用。

請確認資料夾結構是：

```text
extensions/
└─ st-app-interface/
   ├─ manifest.json
   ├─ index.js
   ├─ style.css
   └─ README.md
```

不要多包一層資料夾，否則 SillyTavern 找不到 `manifest.json`。

### 方法二：Git 安裝

把 `st-app-interface` 資料夾上傳到 GitHub／GitLab 後，在 SillyTavern 的「安裝第三方擴充」貼入 Repository URL。

## 如何設定開場選項

1. 在角色卡編輯頁加入主要開場白。
2. 在「替代開場白」加入其他情境。
3. 建立一個新聊天。
4. 點擊右下角手機圖示開啟 APP。
5. 進入「選擇開場」。
6. 到 APP 的「設定」頁，將 `開場 1`、`開場 2` 改成你想顯示的名稱，例如：

   - 兩年炮友
   - 初次見面
   - 久別重逢
   - 自訂開場

「兩年炮友」並非寫死的功能名稱，只是其中一個可以自由修改的按鈕標題。

## 開場切換規則

- 只有聊天內尚未出現玩家訊息時才能切換開場。
- 若故事已開始，擴充會停止切換，避免覆蓋現有劇情。
- 請先使用 SillyTavern 原生的「開始新聊天」，再選擇另一個開場。
- 開場按鈕名稱只保存在介面設定，不會更改角色卡中的開場白內容。

## 備忘錄與音樂

- 備忘錄使用瀏覽器本機儲存，不會送進模型提示詞。
- 音樂頁只播放本機選取的音訊，不會上傳檔案。
- 清除瀏覽器網站資料時，備忘錄可能一併被刪除。

## 相容性

- 建議 SillyTavern 1.17.0 或更新版本。
- 支援單人角色聊天。
- 群組聊天不提供開場切換，但其他頁面仍可開啟。
- 如果看不到右下角 APP 按鈕，請至「擴充功能 → Story App Interface」確認已啟用。

## 自訂外觀

可直接修改 `style.css` 開頭的 CSS 變數。強調色也能在 APP 設定頁調整。

```css
--story-accent: #a82736;
--story-bg: #0e0d13;
--story-text: #f2eef1;
```
